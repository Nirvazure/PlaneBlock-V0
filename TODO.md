### 第四阶段架构（双云·全实时）

数据流与职责划分：

```
┌─────────────┐     HTTP API      ┌──────────────────────┐
│   Vercel    │ ◀───────────────▶ │ 腾讯云 CloudBase      │
│ Next.js API │   (rooms/friends  │ (Auth + 数据库)       │
└──────┬──────┘   invites...)     └──────────────────────┘
       │                                    ▲
       │ 调用 GET/PATCH rooms 读写房间       │
       │ （可选：WebSocket 服务通过 HTTP     │
       │  代理调用，或直接连 CloudBase）     │
       ▼                                    │
┌──────────────────────┐                    │
│ 阿里云轻量服务器      │ ──────────────────┘
│ WebSocket 服务        │   (若需直接读 DB)
│ - 房间状态广播        │
│ - 邀请/好友推送       │
└──────────▲───────────┘
           │
     WebSocket 长连接
           │
┌──────────┴───────────┐
│ 前端 (浏览器)         │
└──────────────────────┘
```

**职责说明：**

| 组件 | 职责 |
|------|------|
| 腾讯云 CloudBase | Auth（手机号登录）、数据库（房间/好友/邀请/战绩） |
| Vercel Next.js API | 业务 API（创建房间、加入、更新状态、好友、邀请） |
| 阿里云轻量服务器 | WebSocket 实时推送（房间状态、邀请、好友请求） |
| 前端 | HTTP 调用 Vercel API + WebSocket 连接阿里云接收推送 |
